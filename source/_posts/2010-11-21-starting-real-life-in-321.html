I've started cracking hard at Ruby (on Rails) and i'm really enjoying it so far, learned to play with Facebook API which is nice and lots of other stuff. 
One thing I really like about the RoR framework is the way they handle developing with a DB and models, instead of setting up the DB upfront.
Which would have meant that you have to remember the changes you made or if you're organized you'll need to manage the DDT files by yourself and hope everyone remembers to do the same.
In RoR you have things called migrations which are instances of changes to the DB where you can create tables, create columns, change them  and delete them. The cool thing is that each migration has a set up and  tear down so you can easly rollback and pick and choose which migrations should run. Each new developer that starts working and needs to set up a local DB just needs to run a smaill task which runs all the migrations one after the other and builds the DB from the changes (Kind off like GIT vs SVN now that I think of it - which is also something new to get in to :) )
